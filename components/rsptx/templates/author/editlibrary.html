Reusing a repository

If you are reusing an existing repository for a second book, this is possible but you need to get the configuration right. Each book built from a single pretext source tree will need its own document-id and will need its own target. The first book is probably fine to use the github repo. But the second book will need to have the Repo Path configured correctly. If the project.ptx file is not in the root folder of the repository then you need to fill in the Path to project.ptx accordingly. Consult the PreTeXt guide for more detail.

The path to the existing repo should be /books/<github_repo_name> Note if your document-id does not match your github repo name then you need to set this in order to get your book to build.

These are all advanced options, so please consult the PreTeXt guide or ask for help to make sure you get set up properly

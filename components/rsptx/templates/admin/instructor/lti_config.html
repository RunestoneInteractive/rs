LTI is Community Supported! It works with various LMS platforms (Canvas, Sakai, Moodle, and Desire2Learn) but you need to know how to configure things on your own LMS. There are two different LTI configurations available: LTI 1.1 (traditional) and LTI 1.3. LTI 1.1 can be configured by the instructor. LTI 1.3 requires the LMS admin to configure the tool, but is more secure and has more features.

Please read the docs first, then feel free to ask for help on the #lti_community_support channel on Discord. If you have it working on another LMS please help us out and contribute to the docs so that others can benefit.

LTI 1.3 Integration

Your LMS administrator will need to configure Runestone as a tool available on your LMS. This will only need to happen one time for your LMS. Once it is configured you can use the tool in any course you have on Runestone. Point them to the Runestone LTI 1.3 documentation. (This means this integration is NOT available if you are using the free Canvas for Teachers or some other hosted LMS that does not have a system administrator you can contact.)

Each Runestone course can only be linked to ONE LMS course via LTI 1.3. You can use multiple different Runestone courses (to make use of multiple books) in one LMS course. If you need to disassociate this Runestone course from any LTI links, you can do so by clicking the button below. Doing so will NOT delete any assignments or grades in the LMS, but will remove the link between the two systems. You can even relink the course to the same LMS course later if you need to.

LTI 1.3 Options

Runestone will try to update assignment due dates. Check to disable automatic updates of assignment due dates from LMS assignment settings.
Runestone will try to update assignment scores in your LMS any time a student completes a problem if that assignment's grades are set to released. Check this to disable this behavior and only sync grades when you trigger it from the grading page.

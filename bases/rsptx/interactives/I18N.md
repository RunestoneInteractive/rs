Internationalization
====================

RunestoneComponents uses `rsi18n`, a small dependency-free internationalization
module at `runestone/common/js/rsi18n.js`. (It replaced the Wikimedia
[jQuery.i18n](https://github.com/wikimedia/jquery.i18n) plugin, which has been
removed.) The catalog format and `msg_*` key conventions are unchanged, so the
migration is mostly mechanical.

`rsi18n` exports:

- `load(messages)` — register a catalog, e.g.
  `load({ en: { msg_foo: "Hello $1" } })`.
- `t(key, ...args)` — look up a key and substitute positional `$1, $2, ...`
  placeholders in one call: `t("msg_foo", "World")` → `"Hello World"`.
  Unknown keys are returned verbatim.
- `setLocale(locale)` / `getLocale()` — the locale otherwise defaults to
  `document.documentElement.lang`, then `navigator.language`, then `en`.

Message loading scripts
-----------------------

In the `js` subfolder of your component's source folder, create a catalog file
per language. The file name should follow the pattern:

_message-bundle-name_`.`_language-code_`.js`

where _message-bundle-name_ starts with the component's name and ends with
`-i18n`. For example, the English catalog for the ActiveCode component is
`activecode-i18n.en.js`. Each catalog imports `load` and registers its
messages:

```js
import { load } from "../../common/js/rsi18n.js";

load({
    en: {
        msg_activecode_play_audio: "Play audio",
    },
});
```

Since messages for different components may be loaded in the same page, message
keys should follow the pattern:

`msg_`_component-name_`_`_given-name_

For example: `msg_activecode_play_audio`

The default language is English. For each _message-bundle-name_ there should be
an English catalog with all message keys defined. For any other language you
may define just a subset of those keys; missing keys fall back to English.

Configure I18N
--------------

In your component's main script file, import the catalogs you support (this
registers them via `load`) and import `t` to use them:

```js
import { t } from "../../common/js/rsi18n.js";
import "./mycomponent-i18n.en.js";
import "./mycomponent-i18n.pt-br.js";
import "./mycomponent-i18n.sr-Cyrl.js";
```

English support is mandatory; always import at least the `.en.js` catalog.

Use message keys instead of hard coded messages
-----------------------------------------------

Use `t(key, ...args)` wherever you would otherwise hard-code a string:

```js
this.submitButton.textContent = t("msg_activecode_check_me");
feedbackArea.innerHTML = t("msg_activecode_passed", count);
```

For book authors
----------------

In `conf.py` of your book project, set the `language` variable to an
appropriate language code.
